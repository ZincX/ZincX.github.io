大护法：只能称得上半个故事

发表于 2017-07-17 | 分类于 日常

写在前面

下午看了大护法，可以说「虎头蛇尾」吧，我是坚持「墙上要是有一把枪，就必须要响一下」的人，故事没有讲好，情节太不完整，槽点太多，这里先政治正确地说一句「作为国产动画这样很棒了！制作人员很努力！」。

然后想吐槽一下，我们现在的国产动画，或者捎上电影也可以，真正缺的是好剧本啊。

太子的人设

电影开始的画风非常赞，画面的构图和上色都非常用心，然而一个中年太子的人设实在是太突兀，太子的脸总让我联想到周星驰电影里的如花，太子设计成这样可能是为了方便下面的各种搞笑桥段，然而一个如此好色的太子，居然没有与山顶上那位有着大胸大长腿又穿着裸露的美女发生任何一点交集，实在是人设的浪费。如此这般，又为何把太子设计成这样的人物形象呢？采用儿童的形象岂不是更适合？儿童的形象更方便与欧阳他孙子的互动戏、跟花生人的追逐场面也可以表现得更灵活有趣、与最后太子的伟光正决策也更搭配。这样看来太子的人设最大的用处还是那些略色情的搞笑桥段，而这点，可能我与导演的笑点不同吧，在那样的画风下，有个这样的太子实在是让我瞬间分心。

追逐戏

电影大部分的时间都在演追逐戏：村里小巷间的追逐、丛林间的追逐、洞穴间的追逐，然而追逐戏太多了，多得有点过了，多到后面的每场追逐都在制造一个尿点。每场追逐都太相似：敌追我跑、敌找我躲、敌打我伤、敌人再强大也斗不过主角光环。

话痨的大护法

整场电影男主都在碎碎念，自言自语还有着僵尸一般的语调，一点语气的波动都没有。走路的时候唠叨、被追着跑的时候唠叨、躲在木板底下的时候还在唠叨，而且唠叨的大都是些无赶紧要的内容，我想导演是想给大护法加一个喜欢吐槽的属性？然而请认真点啊！可以参考银魂里的新八叽，吐槽的时候，一定要把握好语气啊，要有一种超市大减价不是超市的大减价是里面卖的东西的大减价的这种落差感啊。整场电影都在听着男主的没有半点波动的叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨叨，真的烦死了啊！!看到电影最后的报幕里有粤语的配音，不知道粤语的会不会好些，之前在广州的时候看的粤语配音的大圣归来，比普通话的要好一些，希望这部的粤语配音会好一些吧。

MacBook 上利用 hexo + git + Github 搭建博客

发表于 2017-07-15 | 分类于 IT

写在前面

一直想为自己搭建一个博客，然而现有的平台都不尽人意，新浪博客太low，Lofter已经开始走贴图路线，CSDN界面太丑不忍直视，或者自己买域名和服务器搭建？然而感觉 WordPress 的信息还是太重了。于是发现了用 hexo + git + Github 来搭建博客的方法。
这个方法便利、省钱，然而网上教程的质量参差不齐，而且随着 hexo 版本升级，很多操作已经不适用；另一方面，这样的方法搭建起来的博客，更新起来虽然方便，但如果更换电脑的话，更新起来就困难了，这里也介绍利用创建 Github 两个分支的方法，实现在更换电脑后，也能轻松地发表新的博文。

方法

创建的思路

hexo 创建博客的原理，简单来说就是 hexo 将本地的网页源文件进行加工合成，再将生成的网页文件发送到 Github 上,然后生成 Github Page，而如果在 Github 上创建 master 和 Hexo 两个分支，Hexo 分支用来保存网页的源文件， master 分支保存推送的网页，即可实现在更换电脑后，也能轻松地发表新的博文。

创建 Github repository

  • 创建 Github 账号。

    注：将来创建好的博客的地址是 自己的id.github.io，所以这一步要选个好 id（当然到了后面 id 也是可以改的）。

  • 点击页面右上角的加号，选择 New repository。
  • repository name 设置为 自己的id.github.io，description 随便写，如果有钱的话下面的 public/private 可以选择 private，接下来不用管，点击 Create repository。
  • 接下来打开的页面，选择第一项添加 README.md，这是将来为项目添加说明的，此项目相关的内容都可以记在这里。
  • 点击 repository 的 branch，创建 Hexo 分支，并在 setting 里将该分支设为默认分支。

为 Github 添加密钥

  • MacBook上有个 app 叫的「终端」，打开之后输入下面这段，双引号中改成自己注册 Github 时的邮件地址。

    $ ssh-keygen -t rsa -C “e-mail address”

  • 输入后点击回车，然后一路回车键按下去，直到最后提示密钥生成成功，点击回车无反应为止。
  • 最后会生成两个文件 id_rsa 和 id_rsa.pub，我们只需要后面这个文件，文件安装在了默认的文件夹，只需要在终端输入以下指令，即可打开密钥所在的文件夹：

    $ open ~/.ssh

  • 用记事本打开 id_rsa.pub，复制文件内所有的内容。
  • 打开 https://github.com/settings/ssh ，在页面的 Title 随便起个标题，下面 key 的输入框里输入刚刚复制的所有内容，然后点击 Add SHH key.

安装 node.js 和 git

  • node.js 官网 https://nodejs.org/en/ ，通用版和最新版随便挑一个下载，之后一路安装即可。
  • git 官网 https://git-scm.com/downloads ，一般 MacBook 上是有默认安装的，可以在终端用下面指令确认一下,没有的话再下载安装：

    $ git –version

安装 hexo

  • 选好自己想安装博客的文件位置，比如我，想安装在 /Documents/Github/Blog/ 文件夹中（接下来的链接地址都是以我自己的例子来说明，请自行代入）。在终端切换到该文件夹的位置：

    $ cd ~/Documents/Github/Blog

  • 然后在 repository 的页面上，点击右上角的 Clone or download ，可以看到一个链接，复制这个链接 https:// 后面的部分。
  • 然后就可以使用 git 把第二步创建好的 repository 复制到这个文件夹里，在终端使用下面的语句：

    $ git@github.com:你的id/你的id.github.io.git

  • 可以看到 Blog 文件夹下多了一个名为 你的id.github.io 文件夹，文件夹内还有一个 README.md 的文件，这就是你之前在 Github 上创建 repository 的时候，创建的 README.md ，现在你在网页上创建的整个 repository 都已经复制到本地了。
  • 在 你的id.github.io 文件夹下再创建一个文件夹，用来装 hexo，比如 ABC。
  • 在终端用以下指令切换到 ABC 文件夹：

    $ cd ~/Documents/Github/Blog/你的id.github.io/ABC

  • 然后就可以在该文件夹里安装 hexo 了，在该文件夹下依次使用以下指令：

    $ hexo init
    $ npm install
    $ npm install hexo-deployer-git

  • 安装完成后，为了能使 hexo 部署到 Github 上，还需要安装一个插件：

    $ npm install hexo-deployer-git –save

部署博客

  • 打开 ABC 文件夹中的 _config.yml，修改以下的参数，其他的参数可以依个人喜好修改（这里 branch 用的是 master ，即我们会把博客生成的文件推到 master 分支）：

  • 在终端用以下指令切换到 你的id.github.io 文件夹

    $ cd ~/Documents/Github/Blog/你的id.github.io

  • 在该文件夹依次执行以下指令，把该文件夹的内容都推送到 Github 上：

    $ git add .
    $ git commit -m “这里填写改动的备注”
    $ git push origin Hexo //这里把改动推送到 Hexo 分支

  • 在终端切换到 ABC 文件夹：

    $ cd ~/Documents/Github/Blog/你的id.github.io/ABC

  • 在 ABC 文件夹中依次使用以下指令，这里的 g 是 generate，d 是 deploy：

    $ hexo g -d

  • 这样一来，博客就全部部署完毕了，浏览器中打开 http://你的id.github.io 就可以看到自己的博客了。

发表博文

  • 想添加博文，可以直接到 ABC/source/_posts文件夹中添加 Markdown 文件（文件的前几行需要复制 hello-world.md 里的文本），也可以先用终端切换到 ABC 文件夹下，再通过以下指令创建文章：

    $ hexo new “文章的标题”

  • 写文章时，可以先在终端使用以下指令：

    $ hexo server

  • 然后就可以一边改一边随手保存，打开链接 http://localhost:4000/ 一刷新，就可以查看实时的改动效果了。
  • 觉得文章没问题了，就可以用以下指令发送到 Github 上，其他人就可以在你的博客链接里看到最新的动态了：

    $ hexo g -d

  • 更新文章后，还有一个可选的操作，就是将本地的博客文件备份到 Github 上，随时备份是一个好习惯，因为电脑被偷或者中病毒需要重装系统，并不是一个可以预料的事情（当然希望大家永远遇不到这样的事情），如果遇到了，就需要接下来的第8步来恢复，这时，有一个最新的备份是最好的，所以可以养成在发表文章后，使用以下语句的好习惯：

    $ git add .
    $ git commit -m “这里填写改动的备注”
    $ git push origin Hexo //这里把改动推送到 Hexo 分支

更换电脑

  • 如果更换了电脑或者重装了电脑系统，需要把上面提到的安装的步骤全部再走一遍，然后使用以下语句进入之前的 Blog 文件夹：

    $ cd ~/Documents/Github/Blog

  • 然后将之前备份在 Hexo 分支的网页源文件复制到本地

    $ git@github.com:你的id/你的id.github.io.git

  • 在终端使用以下指令进入之前的 ABC 文件夹：

    $ cd ~/Documents/Github/Blog/你的id.github.io/ABC

  • 然后再在该文件夹下依次使用以下指令

    $ npm install
    $ npm install hexo-deployer-git

  • 这样一来，环境就布置好了，就可以再次写博客了。

有关其他

  • 关于一直在 你的id.github.io 和 你的id.github.io/ABC 之间的来回切换：

    很多教程都是直接在 Github 的备份文件夹 你的id.github.io 里安装 hexo，而我的方案是 Github 备份在 你的id.github.io 文件夹里，hexo 安装在 你的id.github.io/ABC 文件夹里，因为我发现 hexo 的新版本，在 你的id.github.io 文件夹里 hexo 是无法创建成功的，于是只好在 你的id.github.io 文件夹下再创建一个文件夹 ABC 用来安装。

  • 有的教程说需要在 你的id.github.io 文件夹下执行语句 npm install hexo:

    但是我发现这一步是重复操作，在 你的id.github.io/ABC 文件夹里执行 npm install 后，已经有 node_modules 了，所以这一步跳过了，而且事实也证明跳过了也没有任何的影响。

合租房的室友

发表于 2017-01-09 | 分类于 日常

上个月27号的时候，突发奇想准备换个住处。已经忘了打开租房APP的契机是什么了，只记得打开之后，看到稍远一些的地方，房租比当时住的要便宜很多。当时住的是3090/月，加上一年3000多的服务费，还有每个月的水电，均摊下来，每个月也有3500左右了，而查到的房间，便宜的有2000出头的。一个人也没有多大必要住太大的房子，当时住的朝南，双人床，有阳台，对于孑然一人的我来说，实在是有太大的空间浪费掉了。与其为一片片空地付房租，不如换到一个空间利用率更高的地方。于是当天选好了房源，预约第二天看房，看房之后，便当场签了合同，当晚便把东西搬个大半，第三天又搬了些零碎的东西，第四天中午看了下之前的房子没有大碍，就算是彻底搬了过来。

感觉搬来这里已经是一个月前的事了，其实也就一周多的时间。

1号房间住了一个妹子，有时男朋友会过来，我搬来的那几天，她整夜整夜地在电话里跟男朋友吵架，每天都吵到一两点，墙的隔音又很差，吵架的内容能听个差不多。一天天的，慢慢也就听懂了她在跟她男朋友吵什么。刚来的那几天，是在吵她男朋友跟其他女生搞暧昧，之后吵则是因为男的答应的跟她一起过元旦，她推掉了一切的聚会，买了一堆他喜欢吃的东西，结果男的临时找了个借口自己玩去了，再之后，则是女的想跟男的结婚，于是春节想去见男方的家长，结果他也推掉了，说自己小姨不喜欢她，春节就不要去了云云。偶然的一次，我进门的时候瞥见过她的男朋友，是那种很呆滞的长相，一脸茫然无神，没想到却能把隔壁的妹子忽悠地团团转，后来听到妹子开心地说准备今年结婚，打算去美国生孩子了。真的这样的男朋友能留到结婚么？

我想租的其实是3号房间，比我的房间贵了100，却大了整整一圈，住的是个男的，但是前天早晨还听到有女的来找他，那时我被吵醒了，看了一眼表，5:10，那时地铁还没通，她是怎么来的？搞不懂，怎么想都搞不懂她是怎么来的，下夜班？之后听到他们洗漱的声音，然后是睡觉，然后是行李箱的声音。洗手间之前放着男的一瓶洗发露，一管洗面奶，昨天已经空了，没有这个男的一点痕迹，房间也没再亮过灯，回家了？这也太早了，搬家了？网上显示房子还是出租状态。

5号房间住着一个妹子，通过微信群看过她的朋友圈，洗手间经常有两副牙刷牙膏，但是只有一份洗面奶洗发水，第二套牙刷牙膏可能是一个短暂居住的人的，是男是女还不清楚，也是在前几天，第二份牙刷的杯子里有了第三把牙刷，是一把电动牙刷，所以可能这三个都是妹子，要是三人里有一个男生，就觉得有些怪怪的了。这个妹子貌似是个设计类的工作，有专门的画板，晚上不加班，总是很早下班，在宿舍里打守望先锋打到很晚。虽然微信里有看到她的这些信息，不过房间里却一直没有见过她。

冰箱里现在都是我和1号房间妹子的，3号房没有往冰箱放过东西，5号房的妹子放了一桶牛奶，仅此而已。

1号房间有独立的卫生间，所以外面的卫生间是我与3、5号房间共用，我留意到上周末之后，马桶边上多了一卷纸巾，放在一个新粘上去的防水盒子里。我搞不懂为什么他们都喜欢把纸巾放在公共的区域，之前在广州的时候室友赵Y也喜欢这样，这样造成的麻烦就是我得在我的洗漱用具那里放一小包纸巾，还得记得随时更换，好让他们相信我没有用他们的纸巾。我当然不会用他们的，在这方面我是极其讲究卫生的，所以我也买了自己用的马桶垫——我留意到那卷纸巾用得特别快，下面纸篓里用过的纸也不像是擦屁股用的，所以只可能是：用来擦马桶垫的了。想不通为什么宁愿一次次用纸，还不保证能够擦得很干净，而不愿买一个马桶垫呢？——也因此我猜测这卷纸是3号房间男生的，5号房间的妹子肯定会买马桶垫的，显而易见，这样一来，我倒也不用担心他担心我用他的纸了，一个心大到用纸擦马桶垫的人，肯定不会考虑我是否会用他的纸的。

驾照 get

发表于 2016-07-31 | 分类于 日常

今天是高温假第二天。

最近发生了太多太多的事情，多到我担心今晚困意到来之前，我能否把这些事情写完。
先是7月12号，第三次考科目三终于通过了。

基本上科目三是从7月初开始重新练的，说实话我其实已经不抱任何希望了。因为之前接连挂了两场考试，每次考试有两次机会，所以我实际上已经挂了4次。每次都是莫名其妙地挂掉，第一次是太过紧张忘了系安全带，第二次是开车快到终点了，前面的大卡车到了厂区门口，突然开始转弯，我自作聪明地迅速换了一个车道躲开了那个大车。结果车上的语音就提示我不合格了，因为「连续两次变更车道」，我这个时候才知道这样做是不符合交规的，之前教练也从来没有跟我讲过这个事情，第二次算是被教练给坑了。第三次更是莫名其妙的挂掉了，因为太紧张，语音说「请打开前照灯」的时候，我就愣了，前照灯是什么灯？难道不是前大灯吗？难道说这是区别于前大灯的另外一个灯？那就是小灯了吧？结果就挂掉了，这才知道前照灯是广州这边对前大灯的称呼，而我在公司问了一圈人，负责车灯设计的同事里听说过「前照灯」这个词的人一个也没有。然后就是第四次，一方面是第三次刚刚挂掉，太紧张，一方面，也是考试的车与教练车差别太大，一般起步的时候，都是轻轻松离合，车开始震了，再松刹车，然后车就启动了，结果考试车的离合器轻轻松开一下，车没震，我再轻轻松一下，车就熄火了，然后就挂掉了。

前面挂的这4次，原因不明不白又躲不开，所以我真的是已经放弃挣扎了，不知道第五次第六次又会以什么原因挂掉。我挂都是紧张惹的祸，然而越补考越紧张，所以越考只会考得越糟。考试那些奇葩的条件，车在路上会不会碰到一些奇葩的情况，都难以预料，要我当作之前什么都没发生，没有挂过，信心满满地参加考试，我是不可能做到的。

教练安排的练车也会按时参加，虽然每次不靠谱的教练都会让我等上一两个小时。但是我已经开始考虑要不要换个城市重新开始，回博山考不知道要多久，估计也就几个月而已。后来又跟赵洋打听他们科室有个人就是科目三挂了买的驾照，价格比学车要便宜得多，而且驾照是货真价实的，就是车管所内部人员帮着改成绩，把不合格的改成合格而已，这么一看倒也释然了，大不了就走点特殊途径而已。

然后12号去考试还是紧张地不行。同行的另外两个人也是挂了好多次的，一个挂了三次，这次是他第四次考试，比我还要紧张，另一个跟我一样挂了两次，前几次也是莫名其妙忘了按喇叭之类的挂掉了。在等着考试的时候腿就一直在抖，自己明明是从来不抖腿的人，考前脚就已经抖得酸掉了，真好奇那些天天抖腿的人小腿肚子是不是特别发达粗壮，看来抖腿也是个技术活呢。考试前跟那个挂了三次的人聊，他说跟他一起学车的人，有个女的方向盘都拿不好，科目三却是一次过了，貌似每个教练教的学员，最先过的总是那个学得不怎么好的，学得好的，往往是一定会挂的那一个。他这么一说，想起教练跟我吐槽说，跟我们一起学车的，陆CF是最菜的，结果也是去年12月就拿到驾照了，我练的是最熟练的，车感最好的，结果已经连挂两次了。陆CF确实非常菜，当初我们一起开始学的驾照，我们都开始学科目二的内容了，他还自己在场地上兜圈子练习拿方向盘，一个转弯下来，手里的方向盘就倒着拿了。这么一看，考科目三，看来真的是三分技术，七分心态啊。

于是开始考试后，我一直告诉我自己我是陆CF（真的对不住他= =），一直在心里默念「不要逞强，不要自由发挥，就像陆CF一样，背过流程死记操作，到了哪里做什么项目就好」。一开始倒也顺利，唯一遇到意外的地方，就是左转进入一条小路后的考试，按照考试的原定内容，指令说要变更车道，那时候我在左边的车道，按照要求变到右边的车道，等有「返回原车道」的指令后，我再回到左边的车道，这样就可以了。结果右侧车道前方停着一辆大卡车，我变更到右侧车道之后，「返回原车道」的指令一直没有给出来，没有指令就转弯，电脑那么蠢，一定会判定我犯规的，但如果我不变更车道，就撞上去了，结果更惨，考试挂了还要付修车费。我只好一咬牙，没等语音提示，提前返回原车道了。当时内心可紧张了，心想这下糟了，又要挂了，天要杀我啊。结果返回原车道又走了几十米后，语音提示「返回原车道成功」，直接跳过那条指令了。心里长舒一口气，也算是躲过一劫。到后面就顺顺利利，一口气考完，听到语音的那句「本次考试结束，考试成绩合格」的话之后，真的兴奋的不得了。这时才发现，我的左手因为一直被空调吹着已经冻得冰凉，连手机都拿不住了。

考完科目三之后，拿驾照就快多了。科目四预约了几次都失败了，结果教练直接打来电话，说车管所那边有几个名额，就在第二天考试，问我有没有时间，于是抓紧时间背了背题，结果科目四考了个满分。

就这样，22号，突然就拿到了证。自由来得太突然，兴奋得几天半夜笑醒。心里最大的负担就这样突然解决了，心里甚至有些空落落的感觉。曾经中午烈日当空闷在车里就练那几十分钟，曾经在大润发旁边等很不守时的教练从白天等到天黑，曾经考前紧张地睡不着觉，然而现在一切都结束了，就像是从山顶滚到山下的过程，滚的路上一直担心脑袋会不会撞到石头，等到了山底，终于停下来了，躺在草地上，眯起眼睛望着天空，终于可以静下心来了。
